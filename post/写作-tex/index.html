TeX 引擎

TeX 宏集/格式(Macros)

  1. plain TeX. plain TeX 是高德纳教授为了方便用户, 自己基于原始的 TeX 定义的 格式, 但实际上 plain TeX 的命令仍然十分晦涩.

  2. latex, L. Lamport(1941年2月7日–)教授开发的基于Plain TeX的排版系统.

  3. pdftex, pdflatex, Hàn Thế Thành 博士在他的博士论文中提出了 pdfTeX 这个 对 TeX 引擎的扩展. 二者最主要的差别就是 pdfTeX 直接输出 pdf 格式文档, 而 TeX 引擎则输出 dvi 格式的文档. pdfLaTeX 这个程序的主要工作依旧是将 LaTeX 格式的文档进行解释, 不过此次是将解释之后的结果交付给 pdfTeX 引擎处理.

  4. xetex, xelatex

  5. luatex, lualatex

  6. uptex, uplatex

  7. latex2e,

  8. bibtex

  9. ctex

注意区分宏包(sty)和宏集(cls, sty), 宏集范围更宽, 这些宏集通常被称为编译方式

Tex 发行版(Distribution)

  1. texlive(cross-platform)

  2. mactex(based on texlive) specific for mac

  3. deprecated on win (不推荐使用, 已淘汰, 已弃用, 废弃)

    1. miktex, 专为windows开发, 作者失联

    2. ctex(中科院吴凌云基于miktex开发, 2012以后停止开发) for windows

      注意: 不要安装ctex套装, 存在严重bug, 并且完全过时, 完全被弃用.

    3. protext(based on miktex), 随着 miktex 更新

Tex 引擎, 格式, 程序, 发行版辨析

本主题参看这篇博客. 清华大学Latextalk第23页也讲到这个问题.

  1. 引擎是TeX的具体实现.

    1. Knuth TeX: 高德纳的原版 TeX, 已经难觅踪影

    2. e-TeX: e表示extended, 是实际上的标准TeX, 几乎所有的后续的 TeX 编译器都 是在 e-TeX 基础上开发的

    3. pdfTEX：直接生成 PDF，支持 micro-typography. 是西文世界最常用的 TeX 编 译器. 通常来说, 西文期刊的模板, 用 pdfTeX 作为编译器编译不会错.

    4. LuaTEX：支持 Unicode，内联 Lua，支持 OpenType. 作为 pdfTeX 正统后继者 出现的. 给 TeX 加入了 Lua 脚本的支持, 同时原生支持 Unicode(CJK文字), 但编译起来效率很低速度很慢.

    5. XeTEX：支持 Unicode、OpenType 与复杂文字编排(CTL). e-TeX 的另一个分支 扩展, 和luatex一样支持直接调用系统里的字体. 目前推荐的中文支持办法, 是 在 XeTeX 引擎上, 使用宏的方式解决的. (xeCJK 宏包)

    6. (u)pTEX: 日本方面推动，生成 .dvi, (支持 Unicode). pTeX 及后续的TeX引 擎, 我们习惯称之为"pTeX 系引擎". p是Publish的缩写. pTeX 系引擎里, 目 前用得最广泛的是e-upTeX, 它可以较好地支持 CJK 字符. 不过由于是日本人 开发的, 所以对日语支持是最好的. 用来撰写中文文档, 跟我们的习惯有些出 入, 需要有丰富的经验才能驾驭. pTeX-ng 则是 pTeX 系引擎的新贵, 它是由 国人 Clerk Ma(马起园/李阿玲) 开发的引擎. 其中ng是"next generation"的 缩写. 从命名来看, 果然还是要秒杀一下我们的邻居的. pTeX-ng 已经相对成 熟. 系列介绍在这里.

    7. ApTEX：底层 CJK 支持，内联 Ruby，Color Emoji.

  2. 格式是TeX的语言扩展(宏集封装).

    由于TeX是一个宏语言, 可以用常用宏构造一个宏集, 将其发布为一个格式 (format), 建立在原来tex语言之上. 常用格式有:

    1. plain TEX: Knuth 同志专用

    2. LATEX: 这是兰伯特搞出来的格式, 使用最为广泛. 排版科技类文章的事实标准

    3. ConTEXt：基于 LuaTEX 实现，优雅、易用（吗？）

    你在某个格式之上写作文档(manuscript)时, 你需要先告诉 TeX 引擎你要载入哪个 格式. 因为格式使用得太频繁了, 所以人们倾向于把格式和引擎打包在一起, 这种 打包我们称之为 程序. 也就是说: 程序: 引擎 + dump 后的格式代码. 这样, 就不用每次都告诉 TeX 引擎这件事情了. 以 LaTeX 格式为例, 将它与上述引擎打 包在一起之后, 我们有了:

    1. e-TeX -> LaTeX

    2. pdfTeX -> pdfLaTeX

    3. LuaTeX -> LuaLaTeX

    4. XeTeX -> XeLaTeX

    5. pTeX -> pLaTeX

    6. upTeX -> upLaTeX

    7. e-upTeX -> e-upLaTeX

    8. pTeX-ng -> pLaTeX-ng

    真正的编译器应该指引擎, 但日常使用中, 我们常常将引擎+格式混在一起说.

    1. 西文文档：pdfLaTeX, XeLaTeX, 或LuaLATEX

    2. 中文文档：XeLaTeX, 或者LuaLatex

    3. 考古者：LaTeX

    4. 好奇心者：LuaLaTeX、pLaTeX-ng

    5. 好奇心爆棚者：pLaTeX、upLaTeX、e-upLaTeX

  3. 发行版

    TeX发行版(Distribution), 又叫Tex系统(system), TeX套装(suite). 将上述所有 引擎, 格式, 宏包(slc和sty), 还有一些的驱动, 辅助工具的集合, 包括TeX编辑器 (比如texlive继承了texworks, ctex继承了winedit, mactex继承了texshop), tex 包管理器(比如texlive的tlmgr, mactex的texlive utility等.). TeX 发行版是用 户真正需要安装的东西. 也就是说, 讨论"新手应该安装什么"的时候. 我们实际上 讨论的是"新手应该安装什么TeX发行版.

    可以这么说: tex发行版就是一个tex排版系统的IDE, 和linux发行版, vs这样的IDE 目的一样. 集成了tex文档开发所需的一切工具和资源.

TeX 基本文档结构

tex基本文档分为两部分: 导言和正文. 导言设定样式, 正文承载内容. 把tex文档看做一个container, 导言就是容器的形式, 正文就是容器的内容.

为了最终生成完整的文档, 一个文档项目有如下文件:

  1. src: tex源文件

  2. fig: 图表文件

  3. bib: 引文

  4. cls: 文类

  5. sty: 宏包

  6. bst: 引文样式

  7. makefile: pdf, clean

  8. versions: draft, rev$, final

  9. README

  \documentclass[a4]{article}
  % 文类, 使用*.cls文件, 例如article, report, book, beamer.
  % []内是选项, 设置纸张, 字号等.

  % 导言, 导入宏包, 设置样式
  \usepackage{graphicx} %引用宏包, *.sty文件
  \graphicspath{{fig/}} %设置

  % 导言中还可以加入\newcommand等全局设置.
  % 导言到此为止

  \begin{document}
  % 这里开始是正文

  %%%%%%%%%%%%%%%
  % 文档外部信息
  \title{}
  \author{}
  \date{}
  \abstract{}

  \maketitle
  %%%%%%%%%%%%%%%

  \section{}

  \begin{equation}
  x^2=\frac{y}{2}
  \end{equation}

  \bibliographystyle{}
  \bibliography{}
  \end{document}

    2. \foo[bar]{arg}: 可选参数放在 []

  3. 环境

  4. 特殊符号需要转义: \%、\$、\&、\textbackslash 等

  5. 连续多个空格 = 单个空格 = 单个换行符

  6. TEX/LaTEX 的语法可以修改

文档

  1. 文档示例

    % 用 UTF-8 编码，命名为 xxx.tex
\documentclass{article}                     % 指明文档类型：文章
 % 导言区：设置文档样式
 \usepackage{amsmath}                        % 调用宏包，实现各种功能
 \newcommand\keyword[1]{\textbf{#1}}         % 自定义命令
 
 \begin{document}
% 正文：套用格式
In quantum mechanics, the \keyword{Schr\"odinger equation} is a
mathematical equation that describes the changes over time of a
physical system in which quantum effects, such as \keyword{wave--%
particle duality}, are significant.
 
% 上面的空行表示分段
In classical mechanics, Newton's second law
($\boldsymbol{F}=m\boldsymbol{a}$) is used to make a\ldots{}
 
Time-dependent Schrödinger equation can be written as  % ö 也能直接用
 \[ i\hbar \frac{d}{dt} |\Psi(t)\rangle = \hat{H} |\Psi(t)\rangle. \]
\end{document}

  2. 文档部件

    1. 标题：\title、\author、\date → \maketitle

    2. 摘要：abstract 环境

    3. 目录：\tableofcontents

    4. 章节：\chapter、§ion、⊂section 等

    5. 文献：\bibliography, \bibliographystyle

  3. 文档划分

    1. 凤头猪肚豹尾：\frontmatter、\mainmatter、\backmatter

    2. 大文件拆分编译：∈clude、∈put

  4. 文本标记 (实质上不需要, 因为违反了内容与格式分离的目的)

    1) 加粗：{\bfseries ...} 或 \textbf{...}
2) 倾斜：{\itshape ...} 或 \textit{...}
3) 字号：\tiny、\small、\large、\Large 等
4) 换行：\\
5) 缩进：\indent
6) 居中：\centering 或 center 环境

    正确或者合理的作法是:

    1. 强调文字（意大利体）：\emph{…}

    2. 摘要（居中，小字号，带有标题）：abstract 环境

    3. 引用（左右边距较大）：quote 或 quotation 环境

    4. 自定义新的命令、环境

TeX Tricks
